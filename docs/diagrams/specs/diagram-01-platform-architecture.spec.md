# Diagram 01: Platform Architecture (Alexandrian Academy)

## Purpose
Depict the end-to-end governed learning loop: evidence → anchors → derivatives → learning objects → governance → evaluation.

## Required boxes (semantic labels)
- Aegis Vault (Immutable Evidence)
- Ariadne’s Thread (Anchors & Spans)
- Mnemosyne Forge (Accessible Derivatives)
- Atlas Codex (Learning Objects: Sandbox/Canon)
- Moirai Ledger (ChangeSets: Governance)
- Oracle of Delphi (Evaluations & Detectors)
- Curriculum Builder (Operational Cadence)
- NBA Engine (Next Best Action Policy)
- Renderers (phone/chat/messaging/email/other)

## Required arrows (must be labeled)
- Evidence ingest: Learner/Teacher → Aegis Vault
- Anchoring: Aegis Vault → Ariadne’s Thread
- Derivation: Aegis Vault + Ariadne’s Thread → Mnemosyne Forge
- Authoring: Mnemosyne Forge + Policy → Atlas Codex (Sandbox)
- Validation: Atlas Codex → Oracle of Delphi
- Promotion: Oracle of Delphi + Review → Moirai Ledger → Atlas Codex (Canon)
- Pathing: Atlas Codex (UnitMap/AssessmentPlan) → NBA Engine → Renderers

## Style constraints
- No vendor or third-party product naming.
- Neutral, community-owned naming only.
- Simple, high-contrast diagram suitable for documentation.
