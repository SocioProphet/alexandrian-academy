# OCW + Coursera-Style Integration Notes (Alexandrian Academy)

## Why this matters
OpenCourseWare and MOOC ecosystems commonly center around:
- videos + transcripts/captions
- readings (PDF/HTML)
- assignments/quizzes
- sequencing + recommendations
- user progress + next-best-action nudges

Our platform’s differentiator is that we bind these to:
- provenance (evidence/anchors)
- jurisdiction constraints (policy)
- pedagogy profiles (Montessori/Waldorf/hybrids)
- omnichannel renderers (phone/chat/messaging/email/other)

---

## Common artifact types → our storage
- Video → Mnemosyne Forge artifact (video) + captions + transcript segments
- Reading → Mnemosyne Forge artifact (HTML/PDF) + extracted text segments
- Assignments → AssessmentPlan entries + artifacts (instructions/rubrics)
- Discussion prompts → UnitMap nodes + renderer-specific action templates

---

## Coursera-like sequencing → NBA Engine
Coursera-style "next unit" or "recommended lesson" is:
- NBA policy output = action contract
- Renderers adapt to channel:
  - phone notification
  - chat message
  - email summary
  - SMS/messaging ping

Each action must be explainable:
- "why this next step?" is backed by standards coverage + mastery evidence + user constraints.

---

## OCW-style publishing
OCW typically expects:
- stable public URLs
- downloadable assets
- clear licensing
We export:
- MIT rights fields
- a public bundle manifest (optional)
- a canonical course index derived from UnitMap

---

## Accessibility minimums
- captions required for videos
- alt text for images
- readable transcript segments (time-aligned)
- accommodations hooks (IEP/504/gifted) expressed via policy parameters
