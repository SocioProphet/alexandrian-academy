# Agent Learning Loop Stack Binding

## Purpose

This document binds Alexandrian Academy's governed learning loop to the broader SocioProphet agent stack.

Alexandrian Academy remains the domain source of truth for curriculum, evidence, pedagogy profiles, jurisdiction constraints, governance changesets, evaluation records, and next-best-action learning objects.

The surrounding stack supplies runtime observation, recall, search, execution evidence, and policy enforcement.

## Stack roles

| System | Role in the learning loop |
| --- | --- |
| Alexandrian Academy | Curriculum objects, evidence anchors, Canon/Sandbox promotion, Oracle evaluations, Moirai changesets, NBA learning actions. |
| Prophet Platform | Runtime APIs, evidence receipts, platform events, conformance checks, operational proposals, and service hosting. |
| AgentPlane | Governed run records, placement decisions, replay artifacts, and evidence-bearing agent execution. |
| memory-mesh | Recall, writeback, learning history, recommendation memory, and learner/agent context retrieval. |
| sherlock-search | Discovery and chat-ops retrieval across curriculum, evidence, proposals, traces, and learning artifacts. |
| lampstand | Local search/indexing for desktop and workspace evidence. |
| ontogenesis | Ontology, SHACL-style gates, semantic promotion, and cross-domain meaning alignment. |
| policy-fabric | Policy-as-code gates, review thresholds, jurisdiction overlays, and autonomy controls. |
| global-devsecops-intelligence | Operations-domain AI4IT signals where learning agents need platform health, CI/CD, or runtime context. |

## Agent learning loop

The loop is deliberately evidence-first:

1. Observe learner, curriculum, environment, and platform signals.
2. Recall relevant history from memory-mesh and search surfaces.
3. Retrieve standards, citations, spans, and prior decisions through Sherlock and Academy anchors.
4. Propose a next learning action through the NBA engine.
5. Evaluate the proposal through Oracle of Delphi and policy-fabric gates.
6. Record the decision as a Moirai ChangeSet or runtime evidence receipt as appropriate.
7. Render the action across channel-specific surfaces.
8. Capture outcome evidence.
9. Update memory, search indexes, and evaluation records.
10. Promote only through governed Sandbox to Canon flow.

## Non-negotiable boundaries

- Learning agents do not bypass Canon gates.
- Learning agents do not convert standing into authority; authority still requires the appropriate power or policy grant.
- Learning agents do not make private or sensitive decisions without jurisdiction-aware policy checks.
- Runtime traces and operational signals are evidence inputs, not curriculum truth by themselves.
- Agent learning history must remain explainable through evidence anchors, evaluation records, and changesets.

## Runtime bridge

Prophet Platform should expose Academy learning-loop records as searchable and inspectable operational artifacts, while Academy remains the semantic owner of curriculum-state transitions.

The first bridge should map:

- Academy NBA records to platform action/proposal surfaces.
- Oracle evaluations to evidence receipts.
- Moirai ChangeSets to promotion and replay references.
- memory-mesh recall records to learning-context evidence.
- Sherlock search results to cited learning-context retrieval.
- AgentPlane run artifacts to reproducible agent task evidence.

## Completion target

A complete integration means a learner or teaching agent can ask why a next learning action was recommended and receive:

- the curriculum object;
- the standards and evidence anchors;
- the learner-context memory references;
- the policy and jurisdiction checks;
- the Oracle evaluation record;
- the Moirai governance state;
- the runtime evidence or AgentPlane replay reference;
- the search/discovery trail used to assemble the recommendation.
