# Broker Reasoning Learning Binding

## Purpose

Alexandrian Academy is the learning, evaluation, and canon-promotion surface for the cross-cloud services broker model.

It does not own broker runtime or policy decisions. It turns validated broker patterns, failures, and remediations into evidence-backed curriculum and evaluations.

## Learning objects

Initial learning object families:

- Cross-Cloud Broker Foundations
- ProviderBinding Review Lab
- Broker Anti-Pattern Detection
- PolicyPlane Decision Lab
- AgentPlane Evidence Lab
- Angel of the Lord Broker Critique

## Inputs

Academy broker-learning inputs include:

- broker doctrine
- service-class and provider-class matrix
- provider-binding examples
- policy decisions and exceptions
- AgentPlane run and replay artifacts
- DevSecOps intelligence findings
- SocioSphere propagation and hardening findings

## Outputs

Academy broker-learning outputs include:

- `AcademyTrainingRecommendation`
- sandbox learning objects
- canon learning objects
- evaluation records
- remediation exercises
- operator and agent certification gates

## Promotion rule

Broker-learning objects should not promote to canon unless they cite evidence, include evaluation records, and map to the current broker standard.

## Design invariant

Alexandrian Academy teaches and evaluates broker reasoning. It must not become the broker runtime or policy-decision plane.
